# Firestore Schema

Cyclot uses Cloud Firestore as its primary NoSQL database. Below is the detailed structure of all collections and documents.

## Collections

### 1. `users`
Profiles for authenticated personnel. Managed primarily by Firebase Auth triggers or initial registration.
*   **Document ID**: Matches Firebase Auth UID
*   **Fields**:
    *   `name` (String): Full name of the user.
    *   `email` (String): User's email address.
    *   `role` (String): Authorization role. Valid values: `employee`, `security`, `admin`.
    *   `createdAt` (Timestamp): Profile creation timestamp.

### 2. `bikes`
Inventory of all bicycles.
*   **Document ID**: Auto-generated by Firestore (`id`).
*   **Fields**:
    *   `bikeId` (String): Display identifier for the bike (e.g. "BIKE-1001").
    *   `isAllocated` (Boolean): True if currently rented out to an employee.
    *   `isDamaged` (Boolean): True if flagged as damaged.
    *   `allocatedTo` (String?): User UID of the current renter. Null if not allocated.
    *   `createdAt` (Timestamp): Record creation timestamp.

### 3. `allocations`
Ledger of all active and past bike allocations. Handled securely via Firestore Transactions to prevent race conditions.
*   **Document ID**: Auto-generated by Firestore.
*   **Fields**:
    *   `bikeId` (String): The ID of the allocated bike.
    *   `employeeId` (String): The UID of the borrowing employee.
    *   `status` (String): Current status. Valid values: `active`, `returned`, `reviewed`.
    *   `allocatedAt` (Timestamp): When the allocation started.
    *   `returnedAt` (Timestamp?): When the bike was dropped off by the employee.
    *   `reviewedAt` (Timestamp?): When security finalized the return.
    *   `conditionOnReturn` (String?): Final condition assessment by security (`undamaged`, `damaged`).

### 4. `notifications`
System alerts pushed to employees (e.g., when a bike return is approved).
*   **Document ID**: Auto-generated by Firestore.
*   **Fields**:
    *   `employeeId` (String): The target recipient UID.
    *   `title` (String): Notification heading.
    *   `message` (String): Body content.
    *   `isRead` (Boolean): Read receipt status.
    *   `createdAt` (Timestamp): Time of alert creation.
